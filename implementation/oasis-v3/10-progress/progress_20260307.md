# progress_20260307

## 今日完成
1. 完成 Admin -> Executor 下发签名闭环：
   - `ExecutorDispatchServiceImpl` 下发 `/invoke` 时增加 `X-Oasis-App-Code`、`X-Oasis-Timestamp`、`X-Oasis-Nonce`、`X-Oasis-Signature`。
   - 签名算法与 Executor -> Admin 保持一致（HMAC-SHA256）。
2. 完成 starter `/invoke` 验签与防重放：
   - 内置 Netty HTTP Server 在入队前校验签名、时间窗、nonce。
   - 新增配置项：
     - `oasis.scheduler.server.verify-invoke-signature`
     - `oasis.scheduler.security.nonce-expire-ms`
     - `oasis.scheduler.security.nonce-max-size`
3. 关键流程注释补齐：
   - 调度下发签名、执行器接收验签、异步执行入队关键逻辑均已补注释。
4. 完成 `dispatch_queue` 异步重试与失败补偿：
   - 首次下发失败入队 `dispatch_queue(PENDING)`。
   - `DispatchRetryService` 异步重试与指数退避。
   - 超过重试阈值后补偿为最终失败并标记 `dispatch_queue(DEAD)`。
5. 完成 `shard_lease` 分片租约优化：
   - 新增 `ShardLeaseCoordinator`：`scheduler_node` 心跳、`shard_lease` 续租与抢占。
   - `ScheduleScannerService` 改为仅扫描当前节点持有分片。
6. 模式逻辑重构为 Spring 策略模式：
   - 调度类型：`CRON / FIXED_DELAY / ONCE`。
   - 路由策略：`ROUND / RANDOM / FAILOVER / BROADCAST`。

## 风险与待办
- 当前 `invoke` 签名依赖 admin 与 executor 的系统时间一致，生产建议统一 NTP。
- 二期可考虑将 nonce 防重放缓存升级为可选 Redis（多实例共享防重放窗口）。
- `shard_lease` 的 drain 主动释放机制待补充（当前主要依赖租约过期接管）。
- `dispatch_queue` payload 版本演进协议待补充（用于未来 OpenAPI 动态任务字段扩展）。
