# IMPLEMENTATION_PLAN_V3

## 目标
构建高可用、高负载的分布式调度平台，基于 Oasis 现有技术栈落地。

## 范围
- 一期：任务管理、调度日志、应用默认告警模板、任务告警覆盖、任务告警事件页、starter（HTTP）
- 二期：OpenAPI、长连接通道（Netty/gRPC streaming）、DAG 启动
- 三期：性能优化与生产级验收

## 已确认原则
1. 保留“应用管理”名称。
2. 一期不做“运维中心”页面。
3. 一期不做独立“告警中心”页面。
4. 告警为“应用模板 + 任务覆盖”。
5. 告警事件从任务入口查看，不放在调度日志页。
6. 应用必须在 Admin 先创建并启用，客户端才允许注册。
7. 任务一期仅控制台维护；二期开放 OpenAPI。
8. 数据表不加 `oasis_` 前缀。

## 里程碑
- M1: 目录归档基线、SQL 基线、核心文档齐备
- M2: 调度内核与下发回调闭环
- M3: 前后端页面与 API 联调完成
- M4: 压测、故障演练、发布手册完成

## 验收指标
- 10 万任务规模
- 日触发 5000 万
- 调度延迟 P95 < 3s
- 客户端下线 3 秒内感知
